## Introduction

Bamboost is a Python package designed for managing scientific simulation data. It provides an organized framework for storing, indexing, and retrieving simulation results, making it easier to work with large-scale computational studies. By integrating filesystem-based storage, SQLite for indexing, and HDF5 for efficient data handling, Bamboost streamlines the workflow for simulation-driven research.

## Features

- **Structured Data Management**: Organize simulations into collections, with metadata storage.
- **HDF5-based Data Handling**: Efficient storage and retrieval of numerical data.
- **Indexing with SQLite**: Quickly search and filter simulation datasets.
- **CLI and Python API**: Interact through the command line or programmatically.
- **MPI Support**: Compatible with parallel computing environments.
- **Remote Access**: Synchronize and view simulation data from different systems.

## Installation

To install Bamboost, use `pip`:
```bash
pip install bamboost
```

To install the latest development version from GitHub:
```bash
pip install git+git@cmbm-ethz.gitlab.com/bamboost
```

To install the latest version of the <a href="https://github.com/zrlf/bamboost-tui">bamboost TUI</a>
```bash
pip install git+git@github.com/zrlf/bamboost-tui
```

### Dependencies

Bamboost requires the following dependencies:

- **Python 3.10+**: Ensure you have a compatible version installed.
- **SQLite**: Used for efficient data indexing.
- **h5py**: For managing HDF5 files.
- **NumPy**: Required for numerical data operations.
- **MPI (optional)**: If working in a parallel computing environment.

To install optional dependencies for MPI support:

```bash
pip install bamboost[mpi]
```

## Project Structure

Bamboost organizes **simulations** into **collections** within structured
directories. A typical project setup might look like this:

<Files>
  <Folder name="Collection_1" defaultOpen>
    <Folder name="simulation_1" defaultOpen>
      <File name="data.h5" />
      <File name="data.xdmf" />
      <File name="additional_file_1.txt" />
      <File name="additional_file_2.csv" />
    </Folder>
    <Folder name="simulation_2" defaultOpen>
      <File name="data.h5" />
      <File name="additional_file_3.txt" />
    </Folder>
  </Folder>
</Files>

Each simulation owns a directory inside a collection. The primary HDF5 file
`data.h5` stores metadata and numerical data. Additional files can be stored
alongside the primary data file.

- **`data.h5`**: Contains simulation metadata, and numerical data in HDF5 format.
